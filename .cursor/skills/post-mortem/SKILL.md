# Post-Mortem

Conduct post-mortem analysis.

## Quick Checklist

When conducting post-mortem:

- [ ] **Timeline** documented
- [ ] **Root cause** identified
- [ ] **Impact** assessed
- [ ] **Action items** created
- [ ] **Follow-up** scheduled

## Post-Mortem Template

### 1. Post-Mortem Structure

```markdown
# Post-Mortem: [Incident Name]

## Summary
Brief description of the incident

## Timeline
- [Time] - Event occurred
- [Time] - Detection
- [Time] - Response started
- [Time] - Resolution

## Root Cause
Primary cause of the incident

## Impact
- Users affected
- Duration
- Business impact

## Action Items
- [ ] Action 1
- [ ] Action 2
```

## Best Practices

### ✅ Good Practices

- Conduct promptly
- Be blameless
- Focus on learning
- Create action items
- Follow up

### ❌ Anti-Patterns

- Don't blame individuals
- Don't skip action items
- Don't ignore follow-up
